# 远程桌面控制项目

## 项目简介

这是一个基于 P2P 技术的远程桌面控制项目，支持实时屏幕共享和远程控制功能。项目采用 h264 编码进行视频传输，具有低延迟、高清晰度的特点。

## 目录结构

```
xmrdc/
├── win-lib264-version/
│   ├── client.py        # 控制端程序
│   ├── controlled.py    # 被控端程序
│   └── cursor.png       # 光标图片
├── .gitignore
├── README.md            # 项目说明文件
└── requirements.txt     # 依赖包列表
```

## 功能特点

- **P2P 连接**：通过打洞技术实现点对点连接，无需端口映射
- **实时屏幕共享**：使用 dxcam 进行屏幕捕获，h264 编码传输
- **远程控制**：支持鼠标和键盘控制
- **分辨率调整**：支持 480p、720p、1080p 分辨率切换
- **全屏模式**：支持全屏显示被控端屏幕
- **深色主题**：美观的深色 UI 界面


## 安装方法

1. 克隆项目到本地

2. 安装依赖包：
   ```bash
   pip install -r requirements.txt
   ```

## 使用方法

### 1. 端口说明

被控端监听本地端口 假设为5667,需要有frp服务映射到公网 暴露后如 any-frp.com:45000
controlled中 TCP_LISTEN_PORT = 5667
client中 TARGET_SERVER_IP= 'any-frp.com' TCP_PORT=45000

### 2. 启动被控端

在被控电脑上运行：
```bash
python win-lib264-version/controlled.py
```

运行后会生成一个 6 位数字的连接口令，显示在界面上。

### 3. 启动控制端

在控制电脑上运行：
```bash
python win-lib264-version/client.py
```

在弹出的界面中输入被控端生成的连接口令，点击"连接"按钮。

### 3. 控制操作

- **分辨率切换**：按 F11 键循环切换分辨率（480p、720p、1080p）
- **全屏模式**：按 F12 键切换全屏模式
- **退出全屏**：在全屏模式下按 ESC 键退出

## 配置说明

### 被控端配置

在 `controlled.py` 文件中可以修改以下配置：

- `TCP_LISTEN_PORT`：TCP 信令服务端口
- `VIDEO_BITRATE`：视频编码比特率
- `MAX_FPS`：最大帧率

### 控制端配置

在 `client.py` 文件中可以修改以下配置：

- `TARGET_SERVER_IP`：FRP 服务器地址
- `TCP_PORT`：FRP 映射的公网端口

## 技术原理

1. **P2P 打洞**：通过 FRP 服务器进行 NAT 打洞，建立点对点连接
2. **视频传输**：使用 dxcam 捕获屏幕，通过 h264 编码压缩后传输
3. **远程控制**：通过 UDP 协议传输鼠标和键盘事件
4. **多进程**：使用多进程进行屏幕捕获，提高性能

## 注意事项

1. 确保被控端和控制端都能访问互联网
2. 确保 FRP 服务器正常运行
3. 对于防火墙，可能需要允许相关端口的访问
4. 建议在局域网内使用，以获得最佳性能

## 系统要求

- Windows 操作系统
- Python 3.11 及以上
- 支持硬件加速的显卡
- 稳定的网络连接

## 故障排除

- **连接超时**：检查网络连接和 FRP 服务器状态
- **画面卡顿**：尝试降低分辨率或关闭其他占用资源的程序
- **无法控制**：检查被控端是否正常运行，以及网络连接是否稳定
